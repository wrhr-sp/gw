# 그룹웨어 봇팀 역할 기준

그룹웨어 봇팀은 **아리아 접수 + 싱드 개발 총괄 + 역할별 개발봇** 구조로 운영한다.

| 표시 이름 | Hermes profile | 역할 | OTA 벤치마킹 기준 |
|---|---|---|---|
| 아리아 | `gw-dev-bot` | 계정사용자 요청 접수, 사용자 보고, 승인 확인, 개발 요청 전달 | 사용자 단일 창구 원칙 |
| 싱드 | `singde` | 개발 메인봇, Kanban 파이프라인 생성, 최종 개발 보고 | 어라 `eora` |
| 기획봇 | `gwplanner` | 요구사항, 정책, 우선순위, 구현 계획 | 현책 `hyeonchaek` |
| 구현봇 | `gwbuilder` | 코드 구현, 버그 수정, 파일 변경 | 공수 `gongsu` |
| 리뷰봇 | `gwreviewer` | 코드/문서/보안/누락 리뷰, PR 검토 | 감결 `gamgyeol` |
| 테스트봇 | `gwtester` | 테스트, 재현, 검증 | 시헌 `siheon` |
| 문서봇 | `gwdocs` | 사용자/운영/개발 문서화 | 해문 `haemun` |
| 운영봇 | `gwops` | systemd, 권한, GitHub 자동화, 로컬 운영 점검 | 진수 `jinsu` |

```text
계정사용자 요청 → 아리아 접수/요약 → 사용자 승인 → 싱드 전달
→ 기획 → 구현 → 리뷰 → 테스트 → 문서화 → 싱드 최종 보고
```

- 아리아는 개발봇이 아니다. 직접 구현하지 않는다.
- 싱드는 개발 총괄봇이다. 역할별 봇에게 나눠 맡긴다.
- 작업봇은 Discord/Telegram 같은 외부 플랫폼 토큰을 갖지 않는다.
