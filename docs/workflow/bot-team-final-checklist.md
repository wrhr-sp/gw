## 호출 테스트 결과
- 싱드(`singde`): 총괄봇/메인봇, 사용자를 직접 상대하는 메인봇으로 응답했다.
- 기획봇(`gwplanner`): 사용자를 직접 상대하는 메인봇이 아니며, 싱드가 필요할 때 활용하는 내부 보조 역할로 응답했다.
- 구현봇(`gwbuilder`): 구현을 맡는 내부 보조 봇이며, 메인봇은 아니라고 응답했다.
- 리뷰봇(`gwreviewer`): 리뷰봇이며, 싱드를 보조하는 내부 보조 역할로 응답했다.
- 테스트봇(`gwtester`): 테스트와 검증을 맡는 내부 보조 역할이며, 메인봇은 아니라고 응답했다.
- 운영봇(`gwops`): 배포·운영봇이며, 싱드가 필요할 때 활용하는 내부 보조 역할로 응답했다.
- 문서봇(`gwdocs`): 문서 정리와 쉬운 설명을 맡는 내부 보조 역할이며, 메인봇은 아니라고 응답했다.

## 현재 운영 방식
1. 사용자는 싱드에게 요청한다.
2. 싱드가 요구를 정리한다.
3. 필요하면 싱드가 내부 역할 관점으로 나눠 처리한다.
4. 결과는 싱드가 다시 합쳐서 쉬운 한국어로 보고한다.
