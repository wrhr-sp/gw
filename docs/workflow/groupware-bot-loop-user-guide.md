# 그룹웨어 봇 루프 사용 안내

## 한 줄 요약

계정사용자 요청은 아리아가 받고 먼저 대장에게 보고한다. 대장이 싱드 전달을 승인하면, 싱드는 다시 대장에게 오케스트레이션 실행 승인을 요청하고 승인 후에만 역할별 봇팀을 움직인다. 대장이 싱드에게 직접 요청한 경우도 싱드는 바로 실행하지 않고 동일하게 실행 승인을 먼저 요청한다.

## 사용자가 기억할 것

1. 계정사용자 요청 접수와 1차 보고는 아리아가 맡는다.
2. 아리아가 싱드에게 넘기는 것은 "전달 승인"이지 "개발 실행 승인"이 아니다.
3. 싱드는 전달받은 요청을 다시 대장에게 승인 요청한다.
4. 대장이 싱드에게 직접 요청해도 싱드는 요약·범위·위험·예상 파이프라인을 먼저 보고한다.
5. 대장이 싱드 단계 실행을 승인한 뒤에만 오케스트레이션을 시작한다.
6. 위험 작업은 별도 승인 전에는 진행하지 않는다.

## GitHub 관련

Git 저장소와 `gh` 인증이 준비되면 아래로 PR/CI 상태를 확인할 수 있다.

```bash
./scripts/gw-ci-status.sh
./scripts/gw-pr-flow.sh --show-status
```

승인된 오케스트레이션 범위 안에서는 PR 생성, CI 확인, merge, branch 삭제까지 GitHub 작업에 포함된다. 내부 스크립트는 안전을 위해 `--approved` 플래그를 붙여 실행한다.
