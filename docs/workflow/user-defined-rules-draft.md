# 그룹웨어 프로젝트 사용자 정의규칙 초안

- 문서 상태: 초안
- 목적: 숙박업 그룹웨어 개발을 시작하기 전에, 사용자가 원하는 작업 방식과 제품 방향을 봇 팀 전체가 따르도록 정리한다.
- 적용 대상: 싱드와 그룹웨어 봇 팀 전체
- 원칙: 이 문서는 `AI_RULES.md`와 `AGENTS.md`에 반영하기 전 사용자 확인을 받는다.

## 1. 최우선 제품 방향

- 지금 만드는 서비스는 일반 숙박 그룹웨어만이 아니라 **기업제휴 기능을 포함한 숙박업 그룹웨어**다.
- 일반 사용자 업무 요청 기능과 함께, 회사/단체/제휴처가 숙박 상품을 이용하는 흐름을 반드시 고려한다.
- MVP 설계에서 기업제휴를 나중 기능으로 완전히 밀어내지 않는다.
- 단, 처음부터 복잡한 기업 정산/계약/승인 시스템을 모두 만들지는 않고, MVP에 들어갈 최소 제휴 흐름을 별도로 정의한다.

## 2. 기업제휴 기능 기본 가정

기업제휴 기능은 아래 범위를 우선 검토한다.

- 기업/단체 계정
- 제휴사별 전용 숙박 요금 또는 할인
- 임직원/회원 인증 방식
- 기업별 업무 요청 가능 정책
- 기업 업무 요청 내역 관리
- 기업 정산 또는 후불 가능성
- 제휴 계약 상태 관리
- 제휴사별 업무 요청 한도 또는 조건

초기 MVP에서는 아래를 우선 후보로 둔다.

- 기업/제휴처 등록
- 제휴 코드 또는 전용 링크
- 제휴 대상자 인증
- 제휴 요금/할인 적용
- 제휴 업무 요청 내역 조회

아래는 후순위로 둔다.

- 복잡한 후불 정산
- 대규모 임직원 권한 관리
- ERP/그룹웨어 연동
- 법인카드 자동 검증
- 제휴사별 맞춤 관리자 포털 고도화

## 3. 개발 우선순위 규칙

개발 순서는 아래 원칙을 따른다.

1. 업무 요청 안정성
2. 조직/업무/객실/재고/요금 구조
3. 일반 사용자 업무 요청 흐름
4. 기업제휴 최소 흐름
5. 결제/취소/환불
6. 운영자 관리 기능
7. 관리자/구성원 관리 기능
8. 고도화 기능

특히 그룹웨어에서 가장 중요한 것은 화면보다 **업무 요청, 재고, 요금, 정책 데이터의 무결성**이다.

## 4. 사용자 소통 규칙

- 사용자는 비개발자이므로 쉬운 한국어로 설명한다.
- 어려운 개발 용어는 짧게 풀어서 설명한다.
- 긴 보고보다 결론, 이유, 다음 선택지를 우선한다.
- 사용자가 "ㄱㄱ", "오케이", "진행"이라고 하면 이미 정한 범위 안에서는 바로 실행한다.
- 애매한 결정이 제품 방향을 바꿀 수 있으면 반드시 선택지를 제시한다.

## 5. 문서 작성 규칙

- 기획 문서, 정책 문서, 보고서는 기본적으로 한국어로 작성한다.
- 개발자가 바로 참고할 수 있도록 파일 경로와 기능 범위를 명확히 쓴다.
- 보고용 긴 설명은 줄이고, 개발 참고 문서에 자세히 남긴다.
- 주요 문서는 `/home/wrhrgw/gw/docs/` 아래에 저장한다.

## 6. 법적·데이터 안전 규칙

- 경쟁사 데이터, 이미지, 문구, 후기, 가격정보를 무단 수집하거나 복제하지 않는다.
- 공개 자료는 구조 학습과 기능 참고에만 사용한다.
- 개인정보, 결제정보, 기업 임직원 정보는 최소 수집 원칙을 따른다.
- 기업제휴 기능은 회사/임직원 정보가 들어갈 수 있으므로 권한, 보관기간, 접근 로그를 설계 단계부터 고려한다.
- 법률 판단이 필요한 영역은 확정하지 않고 전문가 검토 필요로 표시한다.

## 7. 봇 팀 작업 규칙

- 싱드는 총괄로서 사용자와 직접 대화하고 전체 방향을 정리한다.
- 세부 조사, 문서화, 구현, 리뷰, 테스트는 필요한 경우 봇 팀에 배분한다.
- 내부 봇 결과는 검증 없이 최종 사실로 말하지 않는다.
- 완료 보고에는 항상 확인 근거와 남은 리스크를 포함한다.

## 8. 개발 착수 전 필수 문서

개발을 시작하기 전 최소한 아래 문서를 만든다.

1. 그룹웨어 공개자료 시장조사 개발 참고 문서
2. MVP 범위 문서
3. 기업제휴 기능 범위 문서
4. 업무 요청 상태 전이 문서
5. 데이터 모델 초안
6. 화면 목록
7. 취소/환불/정산 정책 초안

## 9. 아직 사용자 확인이 필요한 결정

아래 항목은 확정 전 사용자 확인이 필요하다.

1. 기업제휴의 주 대상
   - 일반 기업 임직원 복지
   - 출장/법인 숙박
   - 협회/단체 회원 혜택
   - 지역 상권/프랜차이즈 제휴

2. 제휴 인증 방식
   - 제휴 코드
   - 회사 이메일 인증
   - 전용 링크
   - 관리자 승인

3. 결제 방식
   - 개인 즉시결제
   - 기업 후불정산
   - 둘 다 지원

4. MVP에서 기업 관리자 페이지를 만들지 여부
   - 없음
   - 최소 조회만
   - 업무 요청/정산까지 포함
