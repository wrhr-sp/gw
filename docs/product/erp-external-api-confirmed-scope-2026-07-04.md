# ERP/경리 외부 API 연결 확정 범위

## 목적

이 문서는 위아히어 그룹웨어의 자체 ERP/경리 기능을 개발한 뒤 마지막 단계에서 연결할 외부 API 범위를 확정해 보관하기 위한 기준 문서다.

현재 원칙은 다음과 같다.

1. 먼저 자체 ERP 내부 기능을 완성한다.
2. 내부 DB/API/UI/RBAC/audit/파일권한/회계 흐름을 안정화한다.
3. 외부 API는 내부 기능이 충분히 돌아간 뒤 마지막 단계에서 연결한다.
4. provider secret, 유료 리소스, production DB 실데이터, 외부 실계정 연동은 연결 시점에 다시 승인받는다.

## 연결 순서

외부 API 연결 순서는 내부 데이터 구조 안정화를 기준으로 한다.

1. 자체 ERP 내부 기능 완성
   - 거래처
   - 지출결의
   - 증빙함
   - 매출/청구
   - 입출금/미수금
   - 계정과목
   - 전표
   - 분개장
   - 원장
   - 마감
   - 부가세
2. 권한/RBAC/audit/개인정보 접속기록 완성
3. 파일 권한/R2/S3/다운로드 로그 완성
4. 외부 연동 공통 event/log/failure/retry 구조 완성
5. sandbox 또는 provider 계약 후 외부 API 연결
6. live smoke는 실계정/실데이터 사용 전 별도 승인으로 진행

## 구현 시 고정 원칙

- 외부 provider가 없어도 내부 ERP 기능은 실제 DB/API/UI로 동작해야 한다.
- 외부 API 미설정 상태에서는 성공처럼 보이는 fallback을 반환하지 않는다.
- provider 미설정은 명시 오류로 처리한다.
  - `EXTERNAL_PROVIDER_NOT_CONFIGURED`
  - `EXTERNAL_PROVIDER_UNAVAILABLE`
  - `WEBHOOK_SIGNATURE_INVALID`
  - `EXTERNAL_EVENT_DUPLICATE`
- 외부 전송 payload, 응답 원문, secret, 토큰은 화면/로그/문서에 노출하지 않는다.
- 안전 요약값과 내부 resource id만 저장한다.
- production DB 실데이터 변경, secret 입력/교체, DNS/custom domain, 유료 리소스는 계속 별도 승인 gate다.
