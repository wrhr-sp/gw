## A. 사용자가 직접 확정한 핵심

- [x] 호텔은 위아히어 법인 아래 내부 지점이다.
- [x] 초기 사용자는 사내 임직원·호텔 소유주·하우스키핑이며 거래처 임직원은 확장 MVP다.
- [x] 하우스키핑은 사람별 독립 사용자유형이고 공용계정이 아니다.
- [x] 호텔 소유주는 호텔당 1명, 계정당 호텔 1곳이다.
- [x] 다른 소유주로 교체할 때 사람 identity는 재사용하지 않고 연결만 교체한다.
- [x] 객실점검·일매출·운영이슈·소유주 문의를 초기 MVP에 포함한다.
- [x] PMS·채널매니저·OTA·청소상태 흐름은 초기 MVP에서 제외한다.
- [x] 파일은 비공개이며 조회·다운로드 권한은 관리자 화면에서 설정한다.
- [x] 사용자는 파일마다 승인요청 없이 설정된 권한으로 즉시 사용한다.

## C. 사용자 명시예외

### 호텔 소유주 1:1

현재 선택은 78점으로 권장기준 미달이지만 조건부로 두 게이트를 통과한 사용자 예외다.

- 호텔 한 곳당 활성 소유주 개인계정 한 명
- 소유주 계정도 호텔 한 곳만 연결
- 공동소유자 동시접근 미지원
- 다호텔 소유주 전환 미지원
- 공용계정·자격증명 공유 금지

- [x] 이 제약을 초기 MVP에서 유지한다.
