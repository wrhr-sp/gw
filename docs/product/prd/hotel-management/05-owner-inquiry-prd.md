# 호텔 소유주 문의 PRD

## 문서 정보

| 항목 | 값 |
|---|---|
| PRD ID | `HOTEL-MVP-050` |
| 상태 | `user_approved` |
| 근거 | `COM-Q-035~038,048,050,053,056,058~060`, `HDRAFT-006,008~011` |

## 목적

호텔 소유주에게 위탁사 문의처와 인앱 문의티켓을 제공하고 접수·담당·답변·종료를 추적한다. 일반 CRM이나 거래처 전체 문의함으로 확대하지 않는다.

## 문의처

호텔별 전화번호·이메일·운영시간을 표시한다. 연락처와 담당그룹 라우팅은 별도 설정이며 화면에 secret이나 내부 계정정보를 노출하지 않는다.

## 문의유형

회사 공통 DB 기준정보:

1. 계약·운영정책
2. 매출·정산
3. 객실·시설
4. 객실점검·운영이슈
5. 계정·권한
6. 기타

사용된 유형은 삭제하지 않고 비활성화하며 문의에는 당시 ID·명칭 스냅샷을 보존한다.

## 상태

```text
접수 → 담당배정 → 답변중 → 답변완료 → 종료
                           └→ 보완요청 → 답변중
종료 후 30일 이내 → 재개
30일 초과 → 새 문의
```

- 답변완료 후 소유주가 종료 또는 보완요청.
- 7일 무응답이면 자동종료.
- 자동종료·재개는 시스템/행위자와 시각을 감사한다.

## 라우팅

- 문의유형 + 호텔별 담당그룹 자동배정.
- 그룹 안에서 개인담당자 지정.
- 설정 없음은 기본 운영문의함으로 이동하며 유실시키지 않는다.
- 미라우팅·담당지연은 관리자 확인대상으로 표시한다.

## 권한

| 사용자 | 권한 |
|---|---|
| 호텔 소유주 | 자기 호텔에서 본인이 생성한 문의·답변·허용첨부 조회, 댓글·보완요청·종료·재개 |
| 문의담당자 | 허용 호텔·유형의 답변 작성 |
| `HOTEL_INQUIRY_ASSIGN` | 담당그룹·개인담당자 지정 |
| 하우스키핑 | 접근 불가 |
| 일반 사내 임직원 | 추가권한 없으면 접근 불가 |

소유주 교체 후 호텔 업무자료는 유지한다. 신규 소유주는 자기 호텔의 진행중 문의와 허용된 과거 문의 이력을 이어서 조회할 수 있다. 과거 문의·댓글의 원작성자와 당시 표시명은 그대로 보존하고, 이전 소유주의 개인 연락처·인증정보는 신규 소유주에게 노출하지 않는다. 첨부 열람·다운로드는 현재 파일권한 설정을 다시 검사한다.

## 데이터 모델 제안

`hotel_inquiry_categories`, `hotel_inquiry_routes`, `hotel_inquiries`, `hotel_inquiry_assignments`, `hotel_inquiry_messages`, `hotel_inquiry_status_history`, `hotel_inquiry_attachments`.

공개 메시지와 내부메모를 분리하고 외부 소유주 응답 schema에 내부메모를 포함하지 않는다.

## 알림

접수·담당배정·답변·보완요청·자동종료·재개에 인앱 알림을 생성한다. PWA 푸시는 동의·유효구독이 있는 경우만 보내며 실패는 문의 저장을 rollback하지 않는다.

## 수용 기준

- 소유주는 다른 호텔·다른 소유주의 문의 존재도 알 수 없다.
- 라우팅 미설정 문의는 기본 운영문의함에 저장된다.
- 7일 자동종료와 30일 재개 제한을 `Asia/Seoul` 업무일/UTC 사건시각 규칙으로 검증한다.
- 내부메모와 사내 개인정보가 소유주 응답에 포함되지 않는다.
- 문의첨부는 비공개 R2와 설정된 조회·다운로드권한을 따른다.
- 모든 상태·담당·답변·파일접근이 감사된다.
