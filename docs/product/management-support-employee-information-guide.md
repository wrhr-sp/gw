# 경영지원팀 사원정보관리 개발 기준

## 목적

사원정보관리는 관리자페이지 기능이 아니라 경영지원팀 인사관리 업무 기능이다. 신규 입사부터 재직, 조직/직무, 서류, 근태·휴가·급여 연결, 퇴사 처리까지의 사원 생애주기를 경영지원팀 업무 기준으로 다룬다.

## 위치

- 화면 위치: `/management-support/hr`
- 상위 기능: 경영지원팀 업무 → 인사관리
- 내부 탭: 사원정보관리

관리자페이지 `/admin/users`는 계정 생성, 계정 상태, 역할/권한, 보안 설정, 감사로그 연결 같은 계정관리 범위만 다룬다.

## 기능 항목 순서

1. 사원 목록: 검색, 필터, 정렬, 사원 선택의 시작점이다.
2. 사원 등록 / 계정 생성: 운영 DB에 사원과 계정을 함께 저장한다.
3. 사원 기본정보: 이름, 이메일, 사번, 입사일, 재직 상태를 다룬다.
4. 조직 / 지점 / 직무: 회사, 부서, 지점, 직책·직급, 발령 이력을 다룬다.
5. 계정 / 역할 / 권한: 계정 유형, 역할, 고위험 권한, 권한 변경 사유를 다룬다.
6. 보안 설정: 비밀번호 변경, 2단계 인증, 로그인 실패, 세션을 다룬다.
7. 근무 / 재직 상태: 휴직, 잠금, 비활성, 퇴사와 권한 회수 후보를 다룬다.
8. 인사 서류 / 계약: 근로계약서, 동의서, 증명서, 인사 발령 문서를 연결한다.
9. 근태 / 휴가 연결: 해당 사원의 근태·휴가 요약과 관련 화면 연결만 제공한다.
10. 급여 연결: 급여 대상 여부와 급여 화면 연결을 제공하고 민감정보는 별도 보안 영역으로 둔다.
11. 업무 접근 / 포털 접근: 기본업무, 부서업무포털, 지점관리포털, 관리자 접근을 보여준다.

## 감사로그 / 변경이력 기준

변경이력과 감사로그는 사원정보관리 상세 탭에 두지 않는다. 계정 생성, 권한 변경, 상태 변경, 잠금/해제, 비밀번호 초기화, 세션 종료 같은 추적은 관리자페이지 `/admin/audit-logs`에서 확인한다.

사원정보관리 화면에는 필요 시 관리자 감사로그로 이동하는 연결만 제공한다.

## 비대상

- 관리자페이지에 사원정보관리 본기능 추가
- 사이드바에 세부 기능을 임의로 추가
- 신규 권한 체계 임의 생성
- production DB 실데이터 입력
- secret, DNS/custom domain, 유료 리소스 변경
