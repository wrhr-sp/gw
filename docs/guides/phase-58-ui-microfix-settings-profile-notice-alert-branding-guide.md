# Phase 58 UI microfix 가이드 — 설정·내 정보·공지/알림·브랜드/포털 표기 정리

## 한 줄 요약
이번 UI microfix에서는 설정 팝업, 공지사항/알림 팝업, 내 정보 설정, 상단 브랜드/포털 표기를 더 짧고 헷갈리지 않게 정리했다.
대장은 live URL에서 `/dashboard` 또는 `/management` 로그인 후 상단 프로필/설정 흐름만 따라가도 대부분을 바로 확인할 수 있다.

## 이 문서가 다루는 범위
- 설정 팝업 안의 섹션 배치 변경
- 공지사항/알림 팝업 버튼 정리
- 내 정보 설정 문구 정리
- 알림 기능 선택과 퇴근 후 알림 설정의 관계 정리
- 상단 브랜드명을 `We'reHere` 로 통일한 내용
- 일반업무포털/경영업무포털 전환 표기 정리
- UAT 확인 순서
- 최종 보고에 넣을 문장

이 문서는 production DB 변경,
실제 알림 발송 서버 교체,
외부 push/메일/SMS 연동,
secret 입력/교체,
DNS/custom domain,
유료 리소스,
migration,
destructive 작업 문서가 아니다.

## 먼저 기억할 8가지
1. 익명 시작점은 계속 `/login` 뿐이다.
2. 이번 변경은 작은 UI 정리다. 로그인/권한/route/API guard 자체를 새로 여는 작업이 아니다.
3. 설정 팝업 안에서 기기별 화면 설정은 일반 설정 목록과 분리됐다.
4. 공지사항/알림 팝업은 `취소`/`저장` 대신 닫기 `X` 중심으로 읽어야 한다.
5. 내 정보 설정의 불필요한 보조 문구는 줄였지만, 기능 자체를 없앤 것은 아니다.
6. 상위 알림 기능이 꺼지면 관련 퇴근 후 알림도 같이 꺼지거나 비활성으로 읽혀야 한다.
7. 브랜드 표기는 `We'reHere` 로 통일한다.
8. 포털 표기는 `일반업무포털` / `경영업무포털` 자체보다, 어느 쪽으로 이동하는지 더 분명히 보이는 구조로 읽어야 한다.

## 바뀐 점 1. 설정 팝업 안의 기기별 화면 설정 분리
### 어떻게 바뀌었나
- 기존에는 일반 설정 목록 안에 PC 사이드바 설정과 모바일 하단탭 설정이 섞여 있었다.
- 이제는 `기기별 화면 설정` 이라는 별도 섹션/박스로 분리해 읽는다.
- 대표 확인 항목은 `PC 사이드바 마지막 상태 기억`, `모바일 하단탭 간결 표시` 다.

### 사용자는 어떻게 읽으면 되나
- 일반 설정은 계정/알림/기본 동작 설정으로 읽는다.
- 기기별 화면 설정은 화면 배치와 표시 방식 설정으로 읽는다.
- 즉, "설정 성격이 다른 항목을 따로 묶었다"고 이해하면 된다.

### UAT 질문
- 설정 팝업을 열었을 때 기기별 화면 설정이 일반 설정과 한 덩어리처럼 보이지 않는가?
- `PC 사이드바 마지막 상태 기억`, `모바일 하단탭 간결 표시` 가 별도 묶음 안에서 보이는가?

## 바뀐 점 2. 공지사항 팝업과 알림 팝업 footer 버튼 제거
### 어떻게 바뀌었나
- 공지사항 팝업과 알림 팝업에서 `취소`, `저장` 버튼을 없앴다.
- 닫기는 우측 상단 `X` 중심으로 유지한다.

### 사용자는 어떻게 읽으면 되나
- 이 팝업은 긴 설정 저장 화면이 아니라, 확인/둘러보기 성격이 더 강하다.
- 따라서 하단 action footer보다 `X로 닫기`가 자연스럽다.

### UAT 질문
- 공지사항 팝업을 열었을 때 하단 `취소`/`저장` 버튼이 보이지 않는가?
- 알림 팝업도 같은 기준으로 정리되어 있는가?
- 닫기 `X`는 그대로 남아 있는가?

## 바뀐 점 3. 내 정보 설정 > 기본 표시 정보 문구 정리
### 어떻게 바뀌었나
다음 보조 문구를 화면에서 제거했다.
- `읽기전용`
- `직접수정불가`
- `항상표시`

### 사용자는 어떻게 읽으면 되나
- 이미 label, input 상태, 화면 구조만으로 충분히 읽히는 내용은 반복해서 적지 않는다.
- 같은 뜻을 여러 번 써서 화면이 시끄럽게 보이지 않게 정리한 것이다.

### UAT 질문
- 기본 표시 정보 영역에서 위 세 문구가 사라졌는가?
- 문구를 지워도 항목 의미를 읽는 데 큰 혼선이 없는가?

## 바뀐 점 4. 내 정보 설정 > 개인정보 표시범위 문구 정리
### 어떻게 바뀌었나
다음 보조 문구를 화면에서 제거했다.
- `고정`
- `이메일과 등록된 내선번호는 버튼은 보이지만 해제할 수 없습니다.`

### 사용자는 어떻게 읽으면 되나
- 바꿀 수 없는 항목이라는 설명을 과하게 반복하지 않고,
- 실제 토글/버튼 상태로 이해하게 정리했다.

### UAT 질문
- 개인정보 표시범위 영역에서 위 문구들이 사라졌는가?
- 화면이 더 짧아졌지만 항목 상태는 여전히 읽을 수 있는가?

## 바뀐 점 5. 알림 기능 선택과 퇴근 후 알림 설정 관계 정리
### 어떻게 바뀌었나
- `알림 받을 기능 선택` 과 `퇴근 후 알림 설정` 이 서로 모순 없이 같이 움직이도록 정리했다.
- 상위 기능 알림이 꺼지면, 해당 퇴근 후 알림도 같이 꺼지거나 비활성으로 읽혀야 한다.
- 긴 예외 설명 대신 `긴급 공지사항` 같은 꼭 필요한 예외만 짧게 유지한다.

### 사용자는 어떻게 읽으면 되나
- 먼저 "어떤 기능 알림을 받을지"를 고르고,
- 그다음 그 기능의 퇴근 후 알림을 볼 수 있다고 이해하면 된다.
- 상위 기능이 꺼져 있는데 하위 퇴근 후 알림만 켜져 있으면 안 된다.

### UAT 질문
- `알림 받을 기능 선택` 과 `퇴근 후 알림 설정` 이 같은 화면에서 함께 보이는가?
- 상위 기능을 껐을 때 하위 퇴근 후 알림이 그대로 활성처럼 남아 있지 않은가?
- `긴급 공지사항` 같은 예외는 짧고 분명하게만 남아 있는가?

## 바뀐 점 6. 브랜드 표기 `We'reHere` 통일
### 어떻게 바뀌었나
- 상단 브랜드/로고 텍스트를 `We'reHere` 로 통일했다.
- 로그인 화면, 일반업무포털 상단, 경영업무포털 전환 문맥에서 같은 표기를 유지해야 한다.

### 사용자는 어떻게 읽으면 되나
- 브랜드명이 화면마다 다르게 보이면 안 된다.
- 작은 따옴표 모양이나 옛 표기가 섞이지 않고 `We'reHere` 한 가지로 읽혀야 한다.

### UAT 질문
- 상단 브랜드가 `We'reHere` 로 보이는가?
- 로그인 뒤 일반 포털/경영 포털 문맥에서도 같은 표기가 유지되는가?

## 바뀐 점 7. 일반업무포털/경영업무포털 전환 표기 정리
### 어떻게 바뀌었나
- 포털명 옆 장식용 아이콘은 줄이고,
- 이동 action 쪽에서 `어느 포털로 이동하는지`가 더 잘 보이게 정리했다.
- 즉, 현재 포털명 자체를 꾸미는 아이콘보다,
  반대 포털로 이동하는 쪽의 의미를 더 분명히 보여 준다.

### 사용자는 어떻게 읽으면 되나
- 지금 내가 어느 포털에 있는지 먼저 읽고,
- 그다음 다른 포털로 이동하는 action 을 분명히 찾으면 된다.
- `일반업무포털로 이동`, `경영업무포털로 이동`처럼 이동 방향이 더 중요하다.

### UAT 질문
- 현재 포털명 옆 장식 아이콘보다 이동 action 의미가 더 잘 보이는가?
- 일반 포털에서는 `경영업무포털로 이동`, 경영 포털에서는 `일반업무포털로 이동`이 자연스럽게 읽히는가?

## 현재 확인 근거
- 구현/리뷰/테스트 체인 기준으로 이번 microfix 범위는 이미 반영됐다.
- parent tester 재검증 기준:
  - focused web tests 21건 통과
  - `pnpm --filter @gw/web typecheck` 통과
  - `pnpm --filter @gw/web build` 통과
  - `pnpm --filter @gw/web build:cf` 통과
  - ad-hoc modal smoke 통과 후 임시 테스트 파일 삭제
- tester가 다시 확인한 핵심 포인트:
  - `We'reHere` 브랜드와 일반/경영업무포털 상단 표기
  - 설정 팝업의 기기별 화면 설정 분리
  - 공지사항/알림 팝업 footer 버튼 제거
  - 내 정보 설정의 불필요 문구 제거
  - 알림 기능 선택과 퇴근 후 알림 설정의 공존 및 모순 제거

중요:
- 이 문서 작업에서는 live URL을 내가 다시 직접 fetch 하거나 테스트를 재실행하지 않았다.
- 따라서 최종 사용자 보고에는 위 parent tester 재검증 근거와 live 직접 확인 메모를 구분해서 적는 편이 정확하다.

## 최종 보고에 넣기 쉬운 문장
- 설정 팝업 안에서 기기별 화면 설정이 일반 설정과 분리돼 더 쉽게 찾을 수 있게 정리됐습니다.
- 공지사항/알림 팝업은 하단 저장형 footer를 빼고 닫기 중심으로 단순화됐습니다.
- 내 정보 설정의 반복 보조 문구를 줄여 화면이 더 짧고 읽기 쉽게 바뀌었습니다.
- 알림 기능 선택과 퇴근 후 알림 설정이 서로 모순 없이 동작하도록 정리됐습니다.
- 상단 브랜드는 `We'reHere` 로 통일했고, 일반업무포털/경영업무포털 전환 표기도 이동 방향이 더 잘 보이게 정리했습니다.
